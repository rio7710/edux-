# 02. 공통 테스트 규칙 및 더미 데이터

## 1) 공통 실행 규칙

1. 툴 기본 자동 입력은 유지, 사용자 이벤트/텍스트 입력만 더미 주입
2. 랜덤 데이터는 고정 seed 사용
3. 각 케이스는 `준비 -> 실행 -> 검증 -> 정리` 4단계로 기록
4. 결과는 `PASS/FAIL/BLOCK`로만 표기
5. 증빙 4종 필수: `URL`, `요청/응답 로그`, `스크린샷`, `DB 결과`

## 2) 더미 데이터 표준

- 역할 계정: `admin`, `operator`, `editor`, `instructor`, `viewer`, `guest`
- 사용자 네이밍: `t_<role>_<nn>` (예: `t_admin_01`)
- 코스: 최소 3개 (`C-A`, `C-B`, `C-C`)
- 공유 상태: `pending`, `accepted`, `rejected` 각 1건 이상
- 강의: 코스별 2개 이상
- 템플릿/문서: 최소 2개

## 3) 상태 초기화 기준

1. 테스트 시작 전 시드/기준 데이터 스냅샷 저장
2. 케이스 종료 후 생성 데이터 정리 또는 테스트 전용 식별자 유지
3. 다음 케이스에 영향 주는 데이터는 반드시 초기화

## 4) 케이스 기록 템플릿

```md
### [ID] 제목
- 구분: 필수(P0) / 확장(P1)
- 준비: 계정, 데이터, URL
- 실행: 단계별 액션
- 검증(UI): 기대값 vs 실제값
- 검증(MCP): 요청/응답 코드/메시지
- 검증(DB): before/after SQL 결과
- 결과: PASS / FAIL / BLOCK
- 증빙: 링크 또는 파일 경로
```

## 5) 테스트 세션 규칙

1. 동일 기능을 역할별로 연속 실행하지 말고 역할 교차 실행
2. 실패 재현은 최소 2회 반복
3. 권한 변경 테스트는 변경 직후와 재로그인 후 모두 확인

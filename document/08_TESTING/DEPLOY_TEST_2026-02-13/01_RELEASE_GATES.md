# 01. 배포 판정 규칙 (Release Gates)

## 1) 등급 정의

- `필수(P0)`: 배포 조건. 100% 통과 필요
- `확장(P1)`: 권장 항목. 통과율 80% 이상 권장
- `Blocker`: 즉시 배포 중단 사유

## 2) Blocker 정의

- 로그인 불가 또는 토큰 갱신 불가
- 권한 우회(메뉴/URL/API 중 1개라도 우회 가능)
- 데이터 무결성 깨짐(중복/오염/orphan 발생)
- `500` 다발 또는 동일 시나리오 재현 가능
- 공유/수락/해제 동기화 불일치

## 3) 하드 게이트

1. `권한 하드 게이트`: 실패 1건이라도 즉시 중단
2. `MCP 품질 게이트`: 평균 `4.0/5.0` 미만 또는 단일 항목 `3.0` 미만이면 중단
3. `DB 무결성`: 중복/orphan/출처 규칙 위반 1건이라도 중단

## 4) 필수 통과 조건

1. P0 항목 전체 `PASS`
2. Blocker `0건`
3. 권한 하드 게이트 `100% PASS`
4. MCP 품질 게이트 `100% PASS`
5. 롤백 절차 검증 완료

## 5) 실패 처리 규칙

1. 실패 항목은 `원인`, `영향도`, `재현 절차`, `임시 우회`, `영구 수정안`을 기록
2. 동일 원인으로 연결된 실패는 묶어서 RCA 작성
3. 재테스트는 수정 후 같은 데이터/같은 절차로 수행

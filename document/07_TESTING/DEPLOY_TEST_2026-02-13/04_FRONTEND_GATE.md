# 04. 프론트(UX/UI) 게이트

## 1) 필수(P0) 체크리스트

| ID | 체크 문항 | 실행 방법 | 합격 기준 |
|---|---|---|---|
| F-M-01 | 역할별 메뉴 노출이 정책과 일치하는가 | 역할별 로그인 후 메뉴 비교 | 비권한 메뉴 미노출/비활성 + 안내문구 |
| F-M-02 | 권한 없는 액션 클릭 시 이유가 명확한가 | 금지 버튼 클릭 | 권한 없음/로그인 필요 메시지 |
| F-M-03 | 로그인 실패 메시지가 원인별로 분리되는가 | 비번 오류/서버 다운 재현 | 인증 실패와 서버 장애 분리 |
| F-M-04 | 폼 검증(필수/형식/길이)이 즉시 동작하는가 | 로그인/등록/공유 폼 입력 | 잘못된 저장 차단 + 필드 에러 |
| F-M-05 | 로딩/재시도 UX가 끊기지 않는가 | 지연/실패 유도 | 무한 로딩 없음, 재시도 경로 제공 |
| F-M-06 | 모바일 360px에서 화면 깨짐이 없는가 | 주요 페이지 모바일 뷰 | 수평 스크롤 최소화, 핵심 버튼 노출 |
| F-M-07 | 키보드만으로 핵심 플로우 수행 가능한가 | Tab/Enter/Esc | 로그인/저장/취소/모달 닫기 가능 |
| F-M-08 | 공유 수락/거절 후 목록이 즉시 갱신되는가 | 기능공유 화면에서 응답 | 상태 카운트/리스트 즉시 반영 |
| F-M-09 | 세션 만료 시 사용자 안내와 복구 경로가 있는가 | 만료 토큰으로 액션 | 재로그인/재시도 경로 제공 |

## 2) 확장(P1) 체크리스트

| ID | 체크 문항 | 실행 방법 | 합격 기준 |
|---|---|---|---|
| F-E-01 | 텍스트 가독성/대비가 충분한가 | 주요 화면 점검 | 판독 어려움 없음 |
| F-E-02 | 긴 텍스트/특수문자 입력 시 레이아웃 안정적인가 | 긴 제목/설명 입력 | 깨짐/겹침/overflow 없음 |
| F-E-03 | 접근성 속성(autocomplete, aria, label)이 충분한가 | 콘솔 경고/DOM 점검 | 치명 경고 없음 |
| F-E-04 | 반복 클릭/중복 제출이 차단되는가 | 저장/초대 버튼 연타 | 중복 요청 방지 |

## 3) 프론트 실패 분류 규칙

1. UX 불편: 기능은 정상, 안내/피드백 부족
2. 기능 실패: 사용자 액션 결과 미반영
3. 권한 실패: 보여주면 안 되는 요소 노출 또는 우회 가능

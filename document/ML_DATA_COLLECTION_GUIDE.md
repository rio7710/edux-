# 강의 교안 ML 데이터 수집 가이드

> 작성일: 2026-02-19
> 목적: AI 기반 원고→교안 자동 변환 서비스 학습 데이터 구축
> 대상: 데이터 수집 담당자

---

## 1. 개요

현재 담당자가 수동으로 수행 중인 **비정제 원고 → 정제 교안 변환 작업**을
AI가 자동으로 처리할 수 있도록 학습 데이터를 수집합니다.

수집한 데이터는 추후 AI 자동 변환 서비스의 학습(Fine-tuning) 및
검색 기반 생성(RAG)에 활용됩니다.

---

## 2. 콘텐츠 위계 구조

모든 교안은 아래 위계 구조를 따릅니다.

```
Course (코스)
└── Lecture (렉처) × N
    └── 대주제
        └── 소주제
            └── 세부주제
                └── 항목 (재귀 가능)
                    └── 항목
                        └── 항목...
                            └── 콘텐츠 타입
```

### 위계 깊이 기준

| 단계 | 설명 | 예시 |
|------|------|------|
| Course | 전체 과정 | 리더십 과정 |
| Lecture | 회차별 강의 | 1강: 리더십의 본질 |
| 대주제 | 강의 내 큰 챕터 | 자기인식 |
| 소주제 | 챕터 내 섹션 | 감정 인식 |
| 세부주제 | 섹션 내 토픽 | 감정의 종류 |
| 항목 | 최소 설명 단위 (재귀) | 긍정 감정 vs 부정 감정 |

> **항목은 하위 항목을 가질 수 있습니다.**
> 콘텐츠 타입(정의/사례 등)은 항상 **가장 말단 항목**에만 붙입니다.

---

## 3. 콘텐츠 타입

항목의 내용은 반드시 아래 타입 중 하나로 라벨링합니다.

| 타입 | 설명 | 블룸 단계 |
|------|------|-----------|
| `[정의]` | 개념 설명, 용어 정의 | 기억 |
| `[해설]` | 풀어서 설명, 의미 부여 | 이해 |
| `[사례]` | 실제 사례, 예시 | 적용 |
| `[근거]` | 데이터, 통계, 연구 출처 | 분석 |
| `[질문]` | 청중 참여 유도, 생각 자극 | 평가 |
| `[실습]` | 활동, 워크시트, 실습 과제 | 창조 |
| `[요약]` | 핵심 정리, 마무리 | 이해 |
| `[전환]` | 다음 주제로 넘어가는 연결 | - |

### MD 작성 예시

```markdown
## 대주제: 리더십의 본질

### 소주제: 리더십 정의

#### 세부주제: 개념 이해

- 항목: 리더십이란
  - [정의] 리더십이란 공동의 목표를 향해 구성원을 이끄는 영향력이다.
  - [해설] 단순한 직위가 아닌 관계에서 발생하는 힘을 의미한다.
  - [사례] A기업 김대표는 위기 상황에서 팀원과 함께 현장에 나서며...
  - [근거] 하버드 리더십 연구(2019)에 따르면 구성원의 87%가...

- 항목: 리더 vs 관리자
  - 항목: 리더의 특성
    - [정의] 리더는 방향을 제시하는 사람이다.
    - [사례] ...
  - 항목: 관리자의 특성
    - [정의] 관리자는 과정을 관리하는 사람이다.
    - [사례] ...

[전환] 이제 리더십의 본질을 이해했으니 자기인식으로 넘어가겠습니다.
```

## 4. 강의 패턴 유형

교안 전체의 흐름 패턴을 아래 중 하나로 분류합니다.

| 패턴 | 설명 | 특징 |
|------|------|------|
| `위계구조형` | 개념을 체계적으로 분류 | 정의→분류→설명 순서 |
| `흐름형` | 시간/단계 순서로 전개 | 도입→전개→마무리 |
| `문제해결형` | 문제 제시 후 해결책 전개 | 문제→원인→해결→결과 |
| `사례중심형` | 사례로 시작해 원리 도출 | 사례→분석→원리→적용 |

## 5. 학습 대상자 분류

| 항목 | 선택지 |
|------|--------|
| **레벨** | 입문 / 중급 / 고급 |
| **직군** | 실무자 / 관리자 / 임원 / 일반 |
| **업종** | 제조 / 서비스 / IT / 금융 / 기타 |
| **연령대** | 20대 / 30~40대 / 50대+ / 전체 |
| **배경지식** | 전공자 / 비전공자 / 혼합 |
| **어휘 수준** | 1(쉬움) / 2(보통) / 3(전문) |

> 같은 원고도 대상자에 따라 어휘와 사례가 달라집니다.
> 반드시 **누구를 위한 교안인지** 기록해주세요.

## 6. 파일 구조 및 저장 방법

### 폴더 구조

```
/training-data
├── /001_리더십_입문_관리자
│   ├── raw.txt          ← 비정제 원고 (변환 전)
│   ├── refined.md       ← 정제된 교안 (변환 후)
│   └── meta.json        ← 메타데이터
├── /002_커뮤니케이션_중급_실무자
│   ├── raw.txt
│   ├── refined.md
│   └── meta.json
```

### 폴더 명명 규칙

```
{번호}_{강의주제}_{레벨}_{직군}
예: 001_리더십_입문_관리자
    002_커뮤니케이션_중급_실무자
```

### meta.json 형식

```json
{
  "id": "001",
  "title": "리더십의 본질",
  "category": "리더십",
  "pattern": "위계구조형",
  "target": {
    "level": "입문",
    "role": "관리자",
    "industry": "제조",
    "age_group": "30~40대",
    "vocab_grade": 2
  },
  "source_format": "PPT",
  "quality": 5,
  "notes": "사례 보강 필요",
  "created_at": "2026-02-19",
  "converter": "담당자명"
}
```

### quality 점수 기준

| 점수 | 기준 |
|------|------|
| 5 | 완벽, 수정 없음 |
| 4 | 양호, 경미한 수정 |
| 3 | 보통, 일부 보강 필요 |
| 2 | 미흡, 상당한 수정 필요 |
| 1 | 불량, 재작업 필요 |

## 7. 원본 파일 형식별 처리 방법

| 형식 | 처리 방법 |
|------|-----------|
| `.hwp` | 한컴에서 직접 TXT로 저장 후 raw.txt로 사용 |
| `.pptx` | 파워포인트에서 텍스트 복사 or 자동 변환 툴 사용 |
| `.docx` | 내용 복사 or 자동 변환 툴 사용 |
| `.pdf` | 텍스트 복사 (이미지 PDF는 별도 처리) |

### 이미지/그림 포함 슬라이드 처리

- 텍스트 중심 슬라이드 우선 처리
- 그림이 핵심인 슬라이드는 **그림 설명을 텍스트로 직접 작성** 후 raw.txt에 포함
- 예: `[그림: 리더십 사분면 - X축: 관계지향, Y축: 과업지향, 4개 영역...]`

## 9. 교육공학 이론 근거

이 가이드의 구조는 아래 교육공학 이론에 기반합니다.

| 이론 | 적용 부분 |
|------|-----------|
| **Bloom's Taxonomy** | 콘텐츠 타입 → 학습 수준 매핑 |
| **Component Display Theory (Merrill)** | 콘텐츠 유형 분류 체계 |
| **Elaboration Theory (Reigeluth)** | 위계 구조 (단순→복잡) |
| **First Principles of Instruction** | 사례 중심 전개 패턴 |
| **Gagné의 9가지 교수 사태** | 콘텐츠 타입 순서 설계 |

## 10. 체크리스트 (건당)

수집 완료 전 아래 항목을 확인하세요.

```
□ raw.txt 저장 완료 (원본 그대로, 수정 없이)
□ refined.md 저장 완료 (위계 구조 + 콘텐츠 타입 라벨 포함)
□ meta.json 저장 완료 (모든 항목 기입)
□ 폴더명 규칙 준수
□ 콘텐츠 타입 라벨 누락 없음
□ 학습 대상자 정보 기입
□ quality 점수 기입
```

## 11. 자주 묻는 질문

**Q. 항목이 너무 깊어지면 어디서 잘라야 하나요?**
A. 실제 강의에서 하나의 설명 단위가 되는 수준까지만 나눕니다. 3~4단계 이상 깊어지면 소주제를 새로 만드는 것을 고려하세요.

**Q. 콘텐츠 타입을 여러 개 붙여도 되나요?**
A. 하나의 문단에 하나의 타입을 권장합니다. 복합적인 내용은 문단을 나눠서 각각 라벨링하세요.

**Q. 원본에 없는 내용을 추가해도 되나요?**
A. raw.txt는 원본 그대로 보존하고, refined.md에서 보강한 경우 meta.json의 notes에 기록해주세요.

**Q. 품질이 낮은 데이터도 저장해야 하나요?**
A. 네, quality 점수를 낮게 기입하고 저장하세요. 낮은 품질 데이터도 학습에 활용될 수 있습니다.

# 08. 실행 순서 및 보고 템플릿

## 1) 권장 실행 순서

1. 권한 하드 게이트(`03`) 실행
2. MCP 품질 게이트(`06`) 실행
3. 백엔드 흐름/동기화(`05`) 실행
4. DB 무결성(`07`) 실행
5. 프론트 UX/UI(`04`) 실행
6. 실패 케이스 재현 및 재검증
7. 최종 보고서 작성 및 승인

## 2) 실행 전 고정 명령

```bash
# 서버 상태
curl http://localhost:7777/health
curl http://localhost:5173

# DB/프리즈마 상태
npx prisma migrate status
npx prisma generate

# 타입/빌드/테스트
npx tsc --noEmit
npm --prefix ui run build
npm test -- --run
```

위 명령 중 하나라도 실패하면 본 테스트 실행을 시작하지 않습니다.

### 통계
- P0 PASS: x / y
- P1 PASS: x / y
- Blocker: x건

### 실패 목록

## 4) 필수 증빙 산출물

1. 권한 매트릭스 결과표(Actual/결과 완료본)
2. MCP 품질 점수표(세 축 평균/최저 점수 포함)
3. DB 점검 SQL 결과 파일
4. 실패 케이스 재현 로그(요청/응답/DB 전후)
5. 최종 테스트 요약 보고서
